# Правила проекта (версия 1)

1. **Отрасль** — технологии (программное обеспечение, ИИ-агенты для управления персоналом).

2. **Глобальный тренд** — Agentic Commerce (делегирование принятия решений и действий ИИ-агентам).

3. **Выбранная гипотеза** — агент-модератор для автоматической оценки эффективности фрилансеров и подготовки уведомлений о расторжении контрактов (гипотеза №12).

4. **Носитель проблемы** — CTO или Tech Lead в продуктовом стартапе (10–50 сотрудников, Seed/Series A), где весь разработческий отдел (15–25 человек) — внештатники из разных часовых поясов.

5. **Частота проблемы** — еженедельный анализ активности; критическое решение об увольнении — раз в 2–4 недели (по окончании спринта).

6. **Острота** — 9/10. Цена ошибки (пропущенный неэффективный исполнитель) — $10–20 тыс. потерь за спринт + угроза срыва инвестиционного раунда.

7. **Ключевое ограничение** — агент не принимает решение самостоятельно, а выдаёт рекомендацию и готовит юридически корректное уведомление для утверждения человеком.
